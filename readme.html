SAP Data Validation Tool

📋 Übersicht

Dieses Tool automatisiert die Validierung von SAP-Tabellendaten zwischen verschiedenen Systemen. Es vergleicht Tabellen über RFC-Verbindungen und generiert automatisch Validierungsskripte in SE16XXL.

⚠️ WICHTIGE SYSTEMANFORDERUNGEN

  • SAP-Fenster: Das Programm funktioniert mit bis zu 3 SAP-Fenstern gleichzeitig
  • Fensterposition: Alle SAP-Fenster MÜSSEN auf maximale Höhe eingestellt sein
  • Monitor: Funktioniert NICHT auf Laptop-Bildschirmen - ein großer Monitor wird benötigt!
  • Die SAP-Fenster müssen vor dem Start des Programms geöffnet und korrekt positioniert sein
  • Je mehr SAP-Fenster Sie verwenden (max. 3), desto schneller erfolgt die Verarbeitung

🚀 Schnellstart

  1. Öffnen Sie 1-3 SAP GUI-Fenster und maximieren Sie deren Höhe
  2. Konfigurieren Sie die Dateien (siehe unten)
  3. Starten Sie main.ahk2
  4. Wählen Sie die zu verwendenden SAP-Sessions aus
  5. Das Tool beginnt automatisch mit der Verarbeitung

🔄 Arbeitsweise

  1. Initialisierung: Das Tool liest die Konfigurationsdateien
  2. Session-Auswahl: Sie wählen die zu verwendenden SAP-Sessions (1-3)
  3. Aufgabenverteilung: Tabellen werden auf Worker verteilt
  4. Batch-Verarbeitung: Je 5 Tabellen pro Worker in Batches
  5. Skript-Generierung: Automatische Erstellung von SE16XXL-Validierungsskripten
  6. Batch-Jobs: Automatisches Starten von Hintergrundverarbeitung

💡 Tipps

  • Verwenden Sie Ctrl + Esc um alle Worker-Prozesse zu stoppen
  • Die Verarbeitung erfolgt in Batches - nach jedem Batch werden die Worker neu gestartet
  • Überprüfen Sie regelmäßig die Log-Dateien auf Fehler
  • Die generierten Skripte werden mit Präfix $TABLENAME_COMP_ gespeichert
  • Verwenden Sie 3 SAP-Sessions für maximale Geschwindigkeit

🛠️ Fehlerbehebung

  • Worker startet nicht: Prüfen Sie, ob SAP GUI Scripting aktiviert ist
  • Tabelle nicht gefunden: Überprüfen Sie den Tabellennamen in databases.cfg
  • RFC-Fehler: Stellen Sie sicher, dass die RFC-Destination existiert und erreichbar ist
  • Syntaxfehler im Skript: Prüfen Sie removed_fields.log für problematische Felder

📋 Overview

This tool automates the validation of SAP table data between different systems. It compares tables via RFC connections and automatically generates validation scripts in SE16XXL.

⚠️ IMPORTANT SYSTEM REQUIREMENTS

  • SAP Windows: The program works with up to 3 SAP windows simultaneously
  • Window Position: All SAP windows MUST be set to maximum height
  • Monitor: Does NOT work on laptop screens - a large monitor is required!
  • SAP windows must be opened and correctly positioned before starting the program
  • The more SAP windows you use (max. 3), the faster the processing

🚀 Quick Start

  1. Open 1-3 SAP GUI windows and maximize their height
  2. Configure the files (see below)
  3. Run main.ahk2
  4. Select the SAP sessions to use
  5. The tool will automatically start processing

⚙️ Configuration Files

📁 databases.cfg

Contains the list of SAP tables to validate. One table per line.

MARA
MARC
MARD
LFA1
EKKO
EKPO

📁 rfc_destinations.cfg

Contains the RFC destinations for system comparisons. One destination per line.

R4PCLNT401_TS
R4PCLNT402_TS
R4PCLNT403_TS

📁 script_template.txt

Template for SE16XXL validation scripts. Usually no changes required.

📊 Log Files

File Description
logs/success_tables.log Successfully processed tables with timestamp and generated script names
logs/failed_tables.log Failed tables with error messages and SAP system messages
logs/removed_fields.log Fields removed during validation (not present in target system)
logs/SAPAutomation_*.log General program execution logs with date
sessions.json Real-time status of worker processes and tasks

🔄 How It Works

  1. Initialization: The tool reads configuration files
  2. Session Selection: You select SAP sessions to use (1-3)
  3. Task Distribution: Tables are distributed to workers
  4. Batch Processing: 5 tables per worker in batches
  5. Script Generation: Automatic creation of SE16XXL validation scripts
  6. Batch Jobs: Automatic launch of background processing

💡 Tips

  • Use Ctrl + Esc to stop all worker processes
  • Processing occurs in batches - workers restart after each batch
  • Regularly check log files for errors
  • Generated scripts are saved with prefix $TABLENAME_COMP_
  • Use 3 SAP sessions for maximum speed

🛠️ Troubleshooting

  • Worker won't start: Check if SAP GUI Scripting is enabled
  • Table not found: Verify table name in databases.cfg
  • RFC error: Ensure RFC destination exists and is reachable
  • Script syntax error: Check removed_fields.log for problematic fields
